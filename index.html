Flagship Project 01

AWS infrastructure as code

AWS Terraform GitHub Actions

A cloud project centered on provisioning and managing AWS resources in a repeatable way instead of clicking through the console manually.

  • Provisioned EC2, S3, and IAM resources using Terraform.
  • Used GitHub Actions to make validation and change flow more consistent.
  • Strengthened how I think about modularity, naming, and safe infrastructure updates.

What it shows: infrastructure thinking, automation habits, and a focus on repeatability over one-off setup.

Flagship Project 02

Serverless contact API

Lambda API Gateway DynamoDB

A lightweight backend that turns a front-end contact form into a real application flow using serverless AWS services.

  • Connected API Gateway, Lambda, and DynamoDB into a simple request pipeline.
  • Practiced thinking about validation, handling input cleanly, and keeping the architecture lean.
  • Built a project that reflects both backend curiosity and practical cloud usage.

What it shows: backend flow, cloud integration, and the ability to build small but useful systems end to end.
